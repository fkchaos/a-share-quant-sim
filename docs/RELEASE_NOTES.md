## 2026-06-20（续2）

### 策略参数优化（v27 最终）
- **v27 全组合参数扫描完成**：48组网格扫描（SL×TP×HOLD×EXT×MOM），全局最优：SL=-0.015 TP=0.03 H=7 E=5 M=0.05，夏普6.585，回撤3.5%
- **关键发现**：参数有交互效应，MOM_THRESHOLD 在组合最优=0.05（非单参数最优0.07）
- **已更新**：strategy_map.py / v27_select.py / strategy_adapter.py 全部同步
- **新增扫描脚本**：`sweep_v27_final.py`（48组全组合）

### 问题记录
- 参数交互效应显著：单参数扫描最优 ≠ 组合最优，必须用全组合验证
- EXT_PNL=3% 与 TAKE_PROFIT=3% 重合，延长逻辑在当前参数下不生效
- **持仓天数计算修正**：从日历天数改为交易日天数（回测用面板索引差值，实盘用上证指数K线）

### 参数扫描过程
1. MOM_THRESHOLD 低值扫描（0.01~0.05）：最优 0.05（夏普6.028）
2. MOM_THRESHOLD 高值扫描（0.05~0.10）：最优 0.07（夏普6.043），趋势收敛
3. SL/TP/持仓天数单参数扫描（14组）：最优 SL=-0.015, TP=0.03, H=7, E=5
4. 全组合扫描（48组）：全局最优 SL=-0.015 TP=0.03 H=7 E=5 M=0.05（夏普6.585）

## 2026-06-16 ~ 2026-06-20

### 架构
- **数据库拆分**：`quant.db` → `quant_stocks.db` + `quant_accounts.db`，股票数据与账户数据彻底隔离
- **回测框架重构**：新增 `strategy_adapter.py` + `wf_runner.py`，回测和模拟盘统一选股/评分/交易逻辑
- **账户-策略分离**：`account_runner.py` 重写为子命令模式（`list`/`create`/`switch`/`run --account-id N`），策略不再硬编码绑定账户

### 功能
- **数据校验**：下载后校验价格>0、成交量>0、成交额≈收盘价×成交量
- **Error Handling**：`run_signal`/`run_execute` 加 try-except，失败写 CRON_STATUS error
- **持仓清理**：`load_account` 自动清理退市/停牌持仓
- **选股池统一**：新增 `get_tradeable_codes()`，统一过滤逻辑

### 问题记录
- **面板顺序 bug 导致 v20c 退役**：`load_panel()` 历史上 tpl[3]/[4]/[5] 对应 high/low/open 全错，修复后 v20c 从 WF 94% → 31%，全量 -67%。旧版好结果完全源于错位假阳性
- **LightGBM 因子过拟合**：5 因子组合 Train IC=0.16，Test IC=0.03，独立使用太弱，不融合 v27

### 策略变动
- v20c 退役
- v11b 暂停（代码保留）
- v31 归档（与 v27 高度重叠）
