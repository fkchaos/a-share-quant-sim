# 龙头战法+概念热度打板策略设计

> **设计日期**: 2026-06-30
> **设计目的**: 实现量化版龙头战法和概念热度打板策略
> **背景**: 基于调研报告，龙头战法10年回测年化24.71%，夏普1.546，全面碾压首板战法

---

## 一、策略概述

### 1.1 龙头战法（v80_dragon_leader）

**核心逻辑**：
- 识别市场连板龙头股（二板及以上）
- 在龙头确认后介入，追求主升浪波段收益
- 多层次动态止损止盈体系

**关键因子**：
- 连板数（streak_count）：连续涨停天数
- 板块地位（sector_rank）：在板块内的涨停排名
- 情绪周期（sentiment_cycle）：市场整体情绪状态
- 封单质量（seal_quality）：封单量/成交额

### 1.2 概念热度打板（v81_concept_heat）

**核心逻辑**：
- 识别强势概念板块（热度前2%）
- 选择板块内近期涨停的个股
- 盘中确认涨幅后介入

**关键因子**：
- 概念热度（concept_heat）：板块3日+15日涨幅排名
- 连续涨停（recent_limit）：近1-4日内有涨停记录
- 流动性（liquidity）：成交额≥500万
- 市值（market_cap）：≥20亿

## 二、数据需求与实现限制

### 2.1 我们的数据
- **日线数据**：OHLCV + 换手率
- **股票池**：zz1800（1586只）
- **执行方式**：收盘价执行

### 2.2 打板策略需要但缺失的数据
- **Tick数据**：逐笔成交、逐笔委托（封单量分析）
- **Level-2数据**：十档行情、逐笔成交
- **分钟K线**：分时强度判断、涨跌速
- **集合竞价数据**：9:15-9:25逐笔
- **龙虎榜数据**：游资席位、买卖金额

### 2.3 日线模拟方案

虽然无法实现实时打板，但可以用日线数据模拟核心逻辑：

#### 龙头战法日线模拟
```python
# 识别连板股
if 今日涨停 and 昨日涨停:
    连续涨停天数 += 1

# 龙头确认（二板及以上）
if 连续涨停天数 >= 2:
    龙头候选池.append(股票)

# 介入时机（二板确认后）
if 龙头候选池:
    选择板块地位最高的股票
    次日开盘买入

# 止损止盈
if 浮盈 >= 5%:
    卖出()  # 止盈
if 浮亏 >= 5%:
    卖出()  # 止损
if 持仓天数 >= 3:
    卖出()  # 时间止损
```

#### 概念热度打板日线模拟
```python
# 识别强势板块
板块涨幅 = 板块内所有股票平均涨幅
板块涨停家数 = 板块内涨停股票数量

# 板块热度排名
if 板块涨幅 > 前2% and 板块涨停家数 > 3:
    强势板块.append(板块)

# 选择板块内涨停股
if 股票 in 强势板块 and 今日涨停:
    概念热度候选.append(股票)

# 盘中确认（用收盘价模拟）
if 收盘价/开盘价 - 1 >= 2%:
    次日买入()

# 止盈止损
if 持仓天数 >= 1 and 浮盈 >= 5%:
    卖出()
```

## 五、风险提示

### 5.1 策略局限性
1. **日线数据限制**：无法实现实时打板，只能模拟
2. **执行滞后**：收盘价执行，错过盘中最佳买点
3. **封单数据缺失**：无法分析封单质量
4. **概念热度滞后**：板块涨幅是滞后指标

### 5.2 2026年打板生态
- 传统无脑打板模式正在消亡
- 监管收紧：独占交易单元被禁止
- 因子衰减极快：可能这周好使，下周就失效
- 策略转向：从打板转向基本面短线+事件驱动

### 5.3 建议
1. **小仓位试水**：用总资金的10-20%尝试
2. **先模拟后实盘**：用模拟盘验证策略，至少跑3-6个月
3. **严格止损**：单笔亏损不超过本金的3-5%
4. **弹性优先**：优先选择20cm弹性票（北交所/创业板/科创板）
