# v47 ML 因子设计方案

> 2026-06-26 创建

## 背景

已有 3 个 ML 方向被证伪（ml_hybrid80、ML因子权重预测、ML regime switching）。
核心失败原因：
1. 标签构造存在未来信息泄露
2. 滚动训练窗口太短导致过拟合
3. 特征工程不充分（仅用原始因子值）

## v47 设计：纯 ML Alpha 因子

### 核心思路

**不用 ML 替代评分系统，而是用 ML 生成一个独立因子**，作为 v39i 的补充输入。

- 输入：价量特征（非因子分数，避免多重共线性）
- 输出：横截面收益率预测 → 转化为因子分数
- 用途：作为第 8 个因子加入 v39i 评分体系

### 标签构造（关键改进）

**之前的问题**：用次日收益率作为标签，存在 T+1 执行延迟导致的未来信息泄露。

**v47 方案**：
- 标签 = T+1 开盘到 T+5 收盘的收益率（匹配实际执行时间）
- 排除标签计算窗口内的特征（严格前视分割）
- 对标签做截面 z-score 化（消除市场收益部分，保留 alpha）

### 特征工程

不用原始因子分数，而是用原始价量数据构造特征：

| 特征 | 计算 | 逻辑 |
|------|------|------|
| mom_5 | 5日动量 | 短期动量 |
| mom_20 | 20日动量 | 中期动量 |
| vol_ratio_5 | 5日量比 | 量能变化 |
| vol_std_20 | 20日波动率 | 风险 |
| reversal_3 | 3日反转 | 超跌反弹 |
| illiq_20 | 20日非流动性 | 规模/流动性 |
| pv_corr_20 | 20日量价共振 | 价量协同 |
| gap_ratio | 开盘跳空 | 情绪 |
| turnover_change | 换手率变化 | 资金流入 |
| high_low_range | 高低幅 | 日内波动 |

### 模型选择

**随机森林**（非 XGBoost）：
- 更不容易过拟合（bagging vs boosting）
- 不需要调参（默认参数即可）
- 训练速度快（rolling window 每日重训）
- 输出稳定性好

### 训练方案

- Rolling window：252 天训练，预测 T+1
- 每 5 天重训一次（非每日，减少过拟合）
- 输出：横截面预测收益率排名 → 归一化为 [0,1] 因子分数

### 验证步骤

1. **IC/IR 分析**：ML 因子与 T+5 收益的 IC 序列
2. **分层回测**：按 ML 因子五分位分组，看收益单调性
3. **WF 集成测试**：v39i + ML 因子 vs v39i 单因子
4. **全市场 WF**：train=252, test=126, step=63

### 预期

- 如果 ML 因子 IC > 0.03 且 IR > 0.3 → 有效
- 如果 IC ≈ 0 或 IR < 0.1 → 放弃 ML 方向
- 即使有效，权重不应超过 0.10（避免过度依赖）
