## 06-16 ~ 06-20 — 架构改造 + 工程加固周

**数据库分离 + 回测框架重构 + 账户-策略解耦 + 工程加固**

- **数据库拆分**：单一 `quant.db` → `quant_stocks.db`（股票池+K线+指标）+ `quant_accounts.db`（账户+持仓+交易日志），彻底隔离
- **回测框架重构**：新增 `strategy_adapter.py` + `wf_runner.py`，统一回测和模拟盘的选股/评分/交易逻辑
  - v27 WF 4/4 正收益，夏普 4.16
  - 面板数据顺序明确定义：`[close, volume, amount, open, high, low]`
- **账户-策略分离**：`account_runner.py` 重写为子命令模式（`list`/`create`/`switch`/`run --account-id N`），策略不再硬编码绑定账户
- **v20c 退役**：面板顺序 bug 修复后策略彻底失效（WF 5/16，全量 -67%），旧版好结果完全源于面板错位假阳性。已清理所有引用
- **v11b 暂停**：cron 禁用，代码保留，随时可恢复
- **LightGBM 因子调研完成**：5 个因子（REV_1D/STD60/ROC60/AMP_TREND/MA60DEV）实现并验证。单因子 IC 微弱，LightGBM 组合 Test IC=0.03（过拟合严重），不融合 v27
- **路径兼容**：所有活跃脚本改为项目相对路径（`__file__` 推断），不再需要 `PYTHONPATH` / `BACKTEST_DATA_DIR`
- **Editable Install**：新增 `pyproject.toml`，`pip install -e .` 一键安装
- **init_project.py**：一键初始化脚本，告别 CSV 依赖
- **96 个过时代码归档**到 `scripts/archive/`
- **数据校验**：`update_daily_data_async.py` 下载后校验价格>0、成交量>0、成交额≈收盘价×成交量，异常数据跳过并告警
- **Error Handling**：`account_runner.py` 的 `run_signal` / `run_execute` 加 try-except，失败时写 CRON_STATUS error，不再静默失败
- **持仓清理**：`load_account` 时自动检查退市/停牌持仓，超过 N 天无交易记录自动清除
- **交易日判断**：信号/执行前增加 `is_trade_day()` 检查，非交易日不运行；严格检查当天数据，不用前一天兜底
- **选股池统一**：新增 `get_tradeable_codes()`，统一回测和模拟盘的板块过滤逻辑（排除 688/689/8/4/2 开头），消除两处不一致
- **表现监控框架**：新增 `nav_compare.py`，模拟盘 vs 回测 NAV 对比（框架就绪，后续完善）
- **v31 归档**：v29→v31 重编号，与 v27 高度重叠，不独立回测

**v27 价量共振策略 WF 通过，成为主力策略**

- **v27 WF 通过**：mom_5>2% + pv_corr_20 价量共振 + gap/illiq/boll，WF 15/100% 正收益，夏普 8.66，已上线账户2
- **v27 向量化加速**：WF 选股 + NAV 计算 30x 加速（20s vs 超时）
- **v11b 向量化重构**：信号生成从分钟级降到 6 秒
- **v11b WF 验证脚本**：完整引擎版
- **v28/v29 新因子测试**：v28 滴水穿石（证伪）、v29 球队硬币（证伪）
- **v20c 参数优化**：WF 扫描最优 SL=-2%/TP=5%/hold=5 天
- **v27 参数更新**：TP=10%→5%，hold=8→5，max_buy=6→4
- **统一配置**：三个账户 TP/SL/hold_days 全部从 strategy_map 读取
