> 创建: 2026-06-29
> 状态: 设计阶段
> 来源: BigQuant 策略调研 + 2026 短线策略设计（6个中的第1个）

## 1. 核心逻辑

**波动率压缩 → 能量蓄积 → 突破释放**

在长时间窄幅震荡的个股中，捕捉放量突破箱体上沿的，跟随之后 3-10 个交易日的趋势段。

**和 v39g 的差异**：
- v39g: 追强（mom>3% + 小市值 + 低流动性），本质是动量+小票
- v58a: 抄突破（波动率压缩后放量突破），本质是波动率状态转换
- 两者信号源不同，理论上互补

## 2. 信号构造

### 2.1 因子定义

| 因子 | 计算 | 方向 | 含义 |
|------|------|------|------|
| `atr_compression` | ATR(20) / ATR(60) | 越低越好 | ATR比率衡量波动率压缩程度 |
| `amplitude_20` | (max(high,20) - min(low,20)) / close | 越低越好 | 20日振幅 |
| `volume_surge` | volume / mean(volume, 20) | 越高越好 | 量比 |
| `breakout_signal` | close > max(high, 20).shift(1) | 0/1 | 是否突破20日高点 |
| `ma_convergence` | std(MA5, MA10, MA20) / mean(close, 20) | 越低越好 | 均线粘合度 |

### 2.2 选股条件（硬筛选）

```
atr_compression < 0.6          # ATR(20) 比 ATR(60) 低40%以上
amplitude_20 < 0.15            # 20日振幅 < 15%
volume_surge > 2.0             # 当日量能 > 20日均量2倍
breakout_signal = 1            # 突破20日高点
close > MA20                   # 站上20日均线
```

### 2.3 评分（软排序）

```
score = -1.0 * rank(atr_compression)        # 压缩越狠越好
      + -1.0 * rank(amplitude_20)            # 振幅越小越好
      + +0.5 * rank(volume_surge)            # 放量越大越好
      + -0.5 * rank(ma_convergence)          # 均线越粘合越好
```

## 5. 实验步骤

### Step 1: IC/IR 分析（1天）
- 每个因子 × 每个持有期（1/3/5/10/20天）
- 判断因子方向和显著性
- 标准: |IC Mean| > 0.03 且 |IR| > 0.3

### Step 2: 单因子 WF 验证（1天）
- 通过 IC 的因子逐个跑 WF
- 确认实战价值

### Step 3: 组合因子 WF（1天）
- 仅保留通过单因子 WF 的因子
- 权重基于 IC 均值数据驱动
- 和 v39g 同条件对比

### Step 4: 参数敏感性（可选）
- 如果通过，扫描关键参数（atr_threshold, volume_surge_threshold）

## 6. 预期与风险

### 预期收益
- 如果有效: 年化 15-25%（BigQuant 类似策略 28-40%）
- 和 v39g 互补: 不同信号源（波动率状态 vs 动量）

### 主要风险
1. **假突破多** — A 股震荡市假突破率可能 > 50%
2. **信号稀疏** — 窄震+放量突破同时满足的天数可能很少
3. **和 v39g 相关性高** — 如果突破后也是小市值票，可能实质重复

### 验证重点
- 和 v39g 选股重叠度（如果 > 70% 说明冗余）
- 假突破率（突破后3日跌回的比例）
- 信号覆盖度（每月能选出几只）
