## Week 3 (2026-06-08(一) ~ 2026-06-14(日))

### 策略
- **v27 参数扫描**：MOM 0.01~0.10（12 值）+ SL/TP/持仓天数（14 组单参数 + 48 组全组合）
  - 全局最优：SL=-0.015, TP=0.03, HOLD_MAX=5, EXT=5, MOM=0.05，夏普 6.585，回撤 3.5%
  - 关键发现：参数有交互效应，单参数最优 ≠ 组合最优
  - 已更新：strategy_map.py / v27_select.py / strategy_adapter.py
  - 新增扫描脚本：`sweep_v27_mom_threshold.py`、`sweep_v27_sltp_hold.py`、`sweep_v27_final.py`
- **v32 分析师预期因子（精简版）**：SUE代理+盈利预测上调代理，WF 13/13(100%)正收益，夏普 7.20。剔除冗余因子后夏普+10%，耗时-38%
- **v33 残差动量（双因子版）**：剥离市场Beta+SMB Beta，WF 11/13(85%)正收益，夏普 6.14。双因子反而不如单因子
- **v35 行业轮动**：市值分组代理行业，WF 13/13(100%)正收益，夏普 7.27。行业动量为零贡献

### 架构
- **数据库拆分**：`quant.db` → `quant_stocks.db` + `quant_accounts.db`
- **回测框架重构**：新增 `strategy_adapter.py` + `wf_runner.py`
- **账户-策略分离**：`account_runner.py` 重写为子命令模式

### 功能
- **数据校验**：下载后校验价格>0、成交量>0、成交额≈收盘价×成交量
- **持仓清理**：`load_account` 自动清理退市/停牌持仓
- **选股池统一**：新增 `get_tradeable_codes()`
- **strategy_adapter 支持环境变量覆盖**：用于参数扫描

### 问题记录
- **v35 相对 v27 无实质提升**：收益+0.89pp，夏普+0.12，统计噪声范围内
- **v33 双因子无效**：A股SMB因子与个股动量高度相关
- **面板顺序 bug 导致 v20c 退役**：修复后 WF 94% → 31%
- **LightGBM 因子过拟合**：Train IC=0.16，Test IC=0.03
- **EXT_PNL 与 TAKE_PROFIT 重合**：延长逻辑在当前参数下不生效
- **持仓天数计算修正**：从日历天数改为交易日天数

### 策略变动
- v20c 退役
- v11b 暂停（代码保留）
- v31 归档（与 v27 高度重叠）
