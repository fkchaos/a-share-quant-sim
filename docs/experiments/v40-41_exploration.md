# v40 因子恶化卖出实验记录

> 日期：2026-06-23
> 状态：❌ 证伪（框架可用）

## 实验目的

在 v39c 的多因子评分体系基础上，增加"因子恶化卖出"能力：
1. 持仓股每日重评分，评分低于阈值 → 卖出
2. 卖出候选中评分恢复的 → 延迟止盈止损（买回）

## 设计思路

### 核心逻辑
```
每日信号流程：
1. 持仓股评分 → 评分 < SELL_THRESHOLD → 卖出候选
2. 卖出候选中评分 > BUY_BACK_THRESHOLD → 延迟卖出（移入 hold_plan）
3. 正常选股 top N → buy_plan
```

### 两种卖出触发模式
| 模式 | 逻辑 | 参数 |
|------|------|------|
| threshold | 评分 < SELL_THRESHOLD 连续 N 天 → 卖出 | SELL_THRESHOLD=0.20, SELL_PENALTY_N=1 |
| momentum | 评分从持仓高点回落 > MOMENTUM_DROP_PCT → 卖出 | MOMENTUM_DROP_PCT=0.20 |

### 新增参数
| 参数 | 值 | 说明 |
|------|-----|------|
| SELL_THRESHOLD | 0.20 | 持仓评分低于此值触发卖出候选 |
| BUY_BACK_THRESHOLD | 0.30 | 卖出候选评分高于此值延迟卖出 |
| SELL_PENALTY_N | 1 | 连续N天低于阈值才确认卖出 |
| SELL_MODE | momentum | 默认使用动量回落模式 |
| MOMENTUM_DROP_PCT | 0.20 | 评分从高点回落超20% → 卖出 |

## 代码变更

| 文件 | 变更 |
|------|------|
| `scripts/strategies/v40_factor_exit.py` | 新建：评分函数 + check_factor_exit + select_stocks_v40 |
| `core/strategy_map.py` | 注册 v40（含 factor_exit_fn） |
| `scripts/backtest/strategy_adapter.py` | 注册 _v40_select + _risk_params |
| `scripts/backtest/wf_runner.py` | _calc_factors 分支 + 交易循环中加入 factor_exit 检查 |
| `scripts/sim/account_runner.py` | _run_signal_impl 中加入 factor_exit 逻辑 |

## 实验结果

### WF 回测对比（2023-01 ~ 2025-12, 1 fold）

| 实验 | 收益 | 夏普 | 回撤 | factor_decay 次数 | 结论 |
|------|------|------|------|-------------------|------|
| v39c baseline | 46.01% | 1.18 | 31.9% | — | 基准 |
| v40 threshold=0.35 | 29.44% | 0.88 | 29.8% | 0 | 评分范围不够（bug） |
| v40 threshold=0.25 | 29.44% | 0.88 | 29.8% | 0 | 同上 |
| v40 threshold=0.20 | -45.37% | -1.85 | 52.7% | 612 | 好股票全被卖 |
| v40 momentum drop=30% | 29.44% | 0.88 | 29.8% | 0 | 阈值太宽松 |
| v40 momentum drop=20% | 29.44% | 0.88 | 29.8% | 多个 | 被风控覆盖 |

### 关键发现

1. **评分范围 bug**：权重 sum=1.0 时，全市场评分 max≈0.32。SELL_THRESHOLD=0.35 永远无法触发。修复：权重 sum 调至 3.0 或将阈值降到 0.20。

2. **WF 模式循环遗漏**：v40 的 factor_exit_fn 仅在 wf_runner full 模式循环中调用，WF 模式循环遗漏。修复后 factor_decay 卖出正常触发（612次），但结果不变。

3. **因子恶化被风控覆盖**：v39c 选出的持仓股在因子评分恶化时，往往也已触发硬风控（STOP_LOSS=-1.5% 或 HOLD_DAYS_MAX=5），factor_decay 卖出是冗余的。

4. **threshold 模式阈值太高**：SELL_THRESHOLD=0.20 在权重 sum=1.0 时把好股票也卖了（v39c 选出的股票评分也在 0.20-0.37 区间波动），导致 -45.37%。

## 根因分析

### 为什么因子恶化不提供独立增量？

v39c 的选股逻辑是动量驱动（mom_5 > 3%），选出的股票特征：
- 短期动量强，但持续性不确定
- 持有期短（max 5 天），硬风控已经足够严格
- 评分恶化 ≈ 动量衰减 ≈ 价格开始下跌 ≈ 即将触发止损

在这种高强度筛选 + 短持有期的组合下，**硬风控已经先于因子恶化卖出行动了**，因子恶化信号没有额外信息量。

### 什么场景下因子恶化卖出可能有价值？
- 更长持有周期（hold_max > 10），让因子恶化先于超时触发
- 更宽松的风控参数（如 SL=-5%），让因子恶化先于止损触发
- 选股逻辑本身不是纯动量（如价值因子/质量因子），评分恶化是更早期的信号

## 框架验证结论

尽管因子恶化卖出策略证伪，但 **框架扩展模式验证通过**：

✅ factor_exit_fn 注册和调用链路完整
✅ sell_penalty_tracker 跨日状态追踪正常
✅ threshold/momentum 双模式切换正常
✅ 与硬风控共存无冲突

后续需要新卖出逻辑时，只需修改 `check_factor_exit()` 函数，不需要改动主流程。

# v40b 实验记录 — 每日无条件轮动

## 实验目的
验证"每日无条件轮动"策略在 A 股市场的有效性：
- 每天卖出持仓中评分最低的 4 只
- 每天买入全市场评分最高的 4 只
- 无硬风控（止损/止盈/持有期限制）

## 策略设计

### 核心逻辑
```python
# 每天执行：
sell_list = 持仓中评分最低的 4 只
buy_list  = 全市场评分最高的 4 只（排除已持有）
```

### 参数
- `SELL_COUNT = 4`
- `BUY_COUNT = 4`
- `NO_HARD_RISK = True`
- 评分权重与 v39c 完全一致（sum=1.0）

### 代码文件
- `scripts/strategies/v40_factor_exit.py` — `select_stocks_v40b()` 函数

## 失败原因分析

### 1. 高换手率带来的交易成本
- 每天换手率接近 100%（卖4买4 = 8次交易/天）
- 每次交易成本：佣金 0.03% + 滑点 0.03% = 0.06%
- 每天交易成本：8 × 0.06% = 0.48%/天
- 年化交易成本：0.48% × 252 = **121%/年**

### 2. 追涨杀跌效应
- 评分最高的股票往往是近期强势股（动量因子权重 0.20）
- 第二天容易回调，买入后立即亏损
- 卖出评分最低的股票时，这些股票可能已经超卖反弹

### 3. 无复利效应
- 无法持有优质股超过 1 天
- 没有"让利润奔跑"的机制
- 每次买入都是全新仓位，无法积累收益

### 4. 与 v39c 对比
- v39c 持仓期平均 5 天，换手率约 40%/月
- v40b 持仓期 1 天，换手率 100%/天
- v39c 夏普 1.18 vs v40b 夏普 -1.963

## 教训

1. **高换手率是 A 股量化的大敌**：交易成本远超过选股优势
2. **追涨杀跌在 A 股无效**：A 股动量效应弱，反转效应强
3. **需要降低换手频率**：如每 3-5 天轮动一次
4. **或加入卖出过滤条件**：如只卖亏损+低分的股票
5. **无条件轮动不如有条件持有**：v39c 的硬风控虽然简单，但有效控制了换手率

## 后续方向

- [ ] v40c：每 3 天轮动一次（降低换手频率）
- [ ] v40d：只卖亏损+低分的股票（有条件卖出）
- [ ] v40e：加入动量过滤（避免追涨杀跌）

## Bug 修复记录

### wf_runner.py 买入循环位置错误
- **问题**：买入循环在 `elif` 块内部，v40b 的 `if` 分支满足时跳过了买入循环
- **修复**：将买入循环从 `elif` 块中提出，放到 `if/elif` 之后对所有策略统一执行
- **影响**：v40b 之前所有 fold 收益率为 0%（未发生任何交易）

# v41 量价共振因子实验记录

## 实验目标
在 v39c 多因子评分体系基础上，新增量价共振因子（VWAP 偏离 + 净支撑量），验证量价因子在 A 股的有效性。

## 实验设计
- **策略版本**：v41
- **新增因子**：
  1. VWAP 偏离（权重 0.15）：close 相对 VWAP 的偏离度
  2. 净支撑量（权重 0.10）：趋势资金日的支撑/阻力成交量差
- **保留 v39c 原有 7 因子**（权重和=1.0）
- **总权重和**：1.25

## 实验结果

### WF 回测（2023-01-01 ~ 2025-12-31）
| 指标 | v39c baseline | v41 | 差距 |
|------|--------------|-----|------|
| 收益率 | +46.01% | +11.90% | -34.11% |
| 夏普 | 1.18 | 0.484 | -0.696 |
| 回撤 | 未知 | 45.17% | - |

### 因子 IC 分析（2023-2025）
| 因子 | IC_5D | ICIR_5D | 结论 |
|------|-------|---------|------|
| size_factor | +0.042 | +0.24 | ✅ 唯一有效 |
| illiq | +0.042 | +0.24 | ✅ 与 size 高度相关 |
| mom_5 | -0.012 | -0.07 | ❌ 反转 |
| turnover_avg | -0.011 | -0.07 | ❌ 反转 |
| vwap_deviation | -0.015 | -0.08 | ❌ 反转 |
| pv_corr_20 | +0.001 | +0.01 | ❌ 无效 |
| fund_flow | 0.000 | 0.00 | ❌ 无效 |
| gap_ratio | NaN | - | ❌ 全零 |
| net_support | NaN | - | ❌ 几乎全零 |

### 交易统计
- 总交易 2007 次，日均近 3 次
- 胜率 21.8%，平均盈利 +14.84%，平均亏损 -25.95%
- 止损占比 58.5%，最大单次亏损 -84.55%

## 失败原因分析

### 1. gap_ratio 因子完全失效
- **根因**：回测框架改为收盘价操作，gap_ratio（开盘跳空）信息无法被利用
- **数据问题**：后复权数据中 open 和 close 同比例调整，差值极小
- **结论**：在收盘价执行框架下，gap_ratio 类因子天然失效

### 2. net_support 因子几乎全零
- **根因**：阈值 `volume > 20日均量 × 1.5` 太严格
- **结果**：90% 以上时间为 0，浪费 10% 权重

### 3. VWAP 偏离 IC 为负
- **根因**：2023-2025 区间 A 股短期反转效应
- **结果**：赋予正权重反而拖累收益

### 4. 止损太松
- **根因**：风控参数未针对新因子调整
- **结果**：平均亏损 -26%，最大 -84.55%，盈亏比仅 0.58

## 教训
1. 收盘价执行框架下不应使用开盘跳空类因子
2. 净支撑量阈值需大幅放宽（建议改为 1.0 倍均量）
3. VWAP 偏离在 2023-2025 为反转因子，不应赋予正权重
4. 止损太松（平均 -26%），应设硬止损（如 -10%）
5. 后复权数据中 open/close 差值极小，gap_ratio 类因子天然失效

## 结论
**放弃 v41，回到 v39c 做参数优化。**

v41 新增因子全部失效，高换手率+宽止损+因子方向错误三重拖累，不如 v39c baseline。
