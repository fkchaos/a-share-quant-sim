# v20 策略调试报告（2026-06-11）

## 问题发现

### 1. v20 WF 失败的根本原因

**CSV 和 DB 数据不一致**，导致涨停计算结果不同：

| 股票 | CSV 涨停次数 | DB 涨停次数 | 差异 |
|------|------------|------------|------|
| 000591（太阳能） | 27 | 14 | +13 |
| 600688 | 13 | 9 | +4 |
| 300750 | 5 | 6 | -1 |
| 600519 | 1 | 3 | -2 |

这种差异直接影响 `recent_limit_up` 因子，导致选股结果完全不同。

### 2. 验证结果

- **CSV 加载**：WF 21.7% / 2.23 夏普 / 15/16 (94%) 正收益 fold ✅
- **DB 加载**：WF 0.1% / 0.00 夏普 / 6/16 (38%) 正收益 fold ❌

### 3. 参数恢复

v20 参数已恢复到原始通过 WF 的值：
- `vol_vs_avg_max = 0.8`
- `range_vs_avg = 0.8`
- `stop_profit = 0.05`
- `hold_days_max = 3`
- `recent_limit_up = 20`

### 4. 三方案对比测试

| 方案 | 年化 | 夏普 | 回撤 | 选股天 |
|------|------|------|------|--------|
| 基线(lu=20d, CSV) | 51.22% | 4.940 | -10.13% | 3998 |
| 方案1(去lu) | -18.4% | -0.836 | -73.3% | 4850 |
| 方案2(lu=60d) | -2.5% | -0.096 | -29.1% | 2056 |
| 方案3(动量替代) | -34.1% | -1.232 | -89.5% | 635 |

**结论**：基线（原始参数 + CSV 加载）是最优的，任何修改都会导致性能下降。

### 5. 待解决问题

1. **CSV vs DB 数据差异**：需要排查为什么 close 数据不一致
2. **v20 不应迁移到 DB 加载**：在数据一致性解决之前，保持 CSV 加载
3. **选股率太低**：1307 天中只有 314 天有信号（24%），2025 年后接近 0

### 6. 建议

- 短期：保持 v20 使用 CSV 加载，参数保持原始值
- 中期：排查 CSV 和 DB 数据差异的根本原因
- 长期：考虑重新设计尾盘选股逻辑，提高选股率
